Recommendation

From the company's point of view, the purpose of introducing the system should be clearly defined, and the fact that the failure of the system can cause great disaster should not be overlooked. Since developing a system requires users to clearly understand what information they want to know through this system, executives must communicate with the development team actively. Although everyone wants the system to be introduced quickly and the company to go well, being fast is not always good. If you want a stable system, you should give the developer enough time.


From the developer's point of view, even if time is running out, this development methodology should be advanced step by step. At each stage, there should be no one in the team who does not know the information. Sufficient time and communication are the most important. There are four steps to successfully implement the ERP system: identification, design, verification, and deployment phase.


  1. Identification

    • Before starting development, create a team member. At this time, third-party, project manager, expansion partner, end-users, and data scientist should exist in the team if possible.
    • Planning the project process.
    • Identifying the company's legacy system where the system will introduced.
    • Identifying user requirements.

  2. Design

    • Compare the leacy system analysis data with the ERP standard system process.
    • Design and develop a new system with analysis data.

  3. Verification

    • Conduct a test to verify that the systems are well integrated.
    • Check if the user contains the desired information.

  4. Deployment

    • The project is not over until the system is stabilised.
    • Prepare alternative measures in advance as there may be problems.
